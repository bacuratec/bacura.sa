# Story 3.1: Requests Filters & Pagination

## Status
Completed

## Story
**As a** user,
**I want** to filter and paginate requests efficiently,
**so that** I can find relevant items quickly without losing my filter state.

## Acceptance Criteria
1. فلاتر الحالة/المدينة/التصنيف تعمل وتتفاعل مع الـ API
2. ترقيم صفحات يحافظ على حالة الفلاتر والتنقل
3. مؤشرات تحميل مرئية دون تجميد الواجهة
4. لا يفقد الفلاتر عند العودة من تفاصيل الطلب

## Tasks / Subtasks
- [x] مزامنة الحالة مع عنوان URL عند الحاجة
- [x] تمرير إجمالي الصفوف لـ pagination server-side في واجهة المستخدم
- [x] إضافة حالة تحميل/خطأ واضحة
- [x] ربط فلاتر المدينة/الخدمة مع endpoint المستخدم (`ordersApi.getUserOrders`)
- [x] زر رجوع يحافظ على الفلاتر باستخدام `router.back()` في التفاصيل
- [x] شرائح فلاتر نشطة قابلة للإزالة لمدينة/خدمة
